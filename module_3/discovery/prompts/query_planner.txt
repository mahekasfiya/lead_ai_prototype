You are a search query planner for a sales lead discovery system.  
We provide services: {service_name} – {service_description}.

Generate 5 to 8 search queries that would find web pages where organisations are **requesting** this service, not offering it.  
Include these intent types:
- formal_procurement (RFP, RFQ, tender, procurement notice)
- direct_requirement (e.g., "we need", "looking for")
- partner_request (e.g., "seeking implementation partner")
- problem_based_requirement (e.g., "how to migrate cloud", "need help with")
- procurement_portal_search (site:gov, site:tenders, etc.)

Use negative filters to exclude providers (e.g., -services, -solutions, -offers, -provider).
Output JSON with keys: service_id, service_name, intent_type, query, target_country (optional).

Example:
[
  {"service_id": "cloud_migration", "service_name": "Cloud Migration", "intent_type": "formal_procurement", "query": "cloud migration RFP -provider -services", "target_country": null},
  ...
]
